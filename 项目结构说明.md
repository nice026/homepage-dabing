# 项目结构说明文档

## 项目概述

这是一个基于 Next.js 15.5.6 构建的个人主页项目，主要展示个人作品集、工具库和联系方式。项目采用 TypeScript 开发，使用 Tailwind CSS 进行样式设计，并集成了 Supabase 作为后端服务。

## 技术栈

- **前端框架**: Next.js 15.5.6 (App Router)
- **UI 库**: React 19.1.0
- **样式**: Tailwind CSS 4
- **后端服务**: Supabase (认证和数据库)
- **开发语言**: TypeScript 5
- **构建工具**: Next.js 内置构建系统

## 前端部分

### 页面结构 (app/)

1. **首页 (app/page.tsx)**
   - 主要展示个人作品集和联系方式
   - 使用 PortfolioSection 和 ContactSection 组件

2. **工具库页面 (app/tools-library/page.tsx)**
   - 展示分类工具列表
   - 包含自媒体运营、日常办公和好站分享三个分类

3. **认证页面 (app/auth/)**
   - 处理 OAuth 认证流程
   - 包含回调处理和退出登录功能

### 组件 (components/)

1. **布局组件**
   - `Navbar.tsx`: 网站导航栏，包含个人简介和导航链接
   - `Footer.tsx`: 网站页脚

2. **页面组件**
   - `PortfolioSection.tsx`: 作品集展示区域
   - `ContactSection.tsx`: 联系方式展示区域
   - `ProfileSection.tsx`: 个人资料展示区域
   - `SkillsSection.tsx`: 技能展示区域
   - `AchievementsSection.tsx`: 成就展示区域

3. **认证组件**
   - `GoogleSignInButton.tsx`: Google 登录按钮
   - `GitHubSignInButton.tsx`: GitHub 登录按钮
   - `UserProfile.tsx`: 用户资料组件

## 后端部分

### API 路由 (app/auth/)

1. **认证回调 (app/auth/callback/route.ts)**
   - 处理 OAuth 认证回调
   - 交换授权码获取会话

2. **退出登录 (app/auth/signout/route.ts)**
   - 处理用户退出登录请求

### 数据库与认证 (lib/supabase/)

1. **客户端配置 (lib/supabase/client.ts)**
   - 创建浏览器端 Supabase 客户端
   - 包含环境变量检查

2. **服务端配置 (lib/supabase/server.ts)**
   - 创建服务端 Supabase 客户端
   - 处理 Cookie 和会话管理

### 中间件 (middleware.ts)

- 处理所有请求的中间件
- 刷新用户会话
- 可以扩展添加受保护路由逻辑

## 静态资源 (public/)

- 网站图标和清单文件
- 个人头像图片
- 社交媒体二维码图片

## 开发与部署

### 开发脚本

- `npm run dev`: 启动开发服务器 (使用 Turbopack)
- `npm run build`: 构建生产版本 (使用 Turbopack)
- `npm start`: 启动生产服务器

### 环境变量

项目需要配置以下环境变量：

- `NEXT_PUBLIC_SUPABASE_URL`: Supabase 项目 URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`: Supabase 匿名密钥

## 功能特点

1. **响应式设计**: 适配不同屏幕尺寸
2. **暗色模式**: 支持明暗主题切换
3. **OAuth 认证**: 支持 Google 和 GitHub 登录
4. **工具库**: 分类展示实用工具
5. **SEO 优化**: 包含元数据和网站清单

## 总结

这是一个典型的 Next.js 全栈应用，前端使用 React 组件构建用户界面，后端通过 Supabase 提供认证和数据库服务。项目结构清晰，遵循 Next.js App Router 的最佳实践，适合作为个人主页或作品集展示网站。
